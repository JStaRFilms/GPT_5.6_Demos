# Task 02 — Showcase Experience

## Objective
Build the public-facing gallery and experiment viewing experience.

## Scope
- Create a distinctive editorial/laboratory visual system with model-specific accents.
- Build home gallery with search/filtering and project status.
- Build model and project routes.
- Build sandboxed iframe preview with reload, viewport, full-screen, and open controls.
- Build a lazy-loaded Pi-inspired transcript viewer with role hierarchy and collapsible tool/process entries.
- Build a prompt-group comparison route for projects represented by multiple models.
- Implement responsive navigation, loading, empty, and error states.

## Definition of Done
- Gallery clearly represents Sol, Terra, and Luna.
- Complete projects are navigable and interactive.
- Project pages work on desktop and mobile.
- Nitro League comparison is available.
- Keyboard navigation, focus states, semantic structure, and motion preferences are handled.

## Expected Artifacts
- App routes, components, styles, and metadata.
- Preview and transcript client components.
- Comparison experience.
