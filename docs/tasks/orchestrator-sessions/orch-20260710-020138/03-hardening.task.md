# Task 03 — Hardening and Deployment

## Objective
Make the showcase maintainable and Vercel-ready.

## Scope
- Add safe iframe policies and content handling.
- Add validation/verification commands and helpful failure messages.
- Ensure large transcript payloads are fetched only on demand.
- Add Vercel-compatible configuration and production metadata.
- Document project ingestion, metadata overrides, transcript publication policy, local development, and deployment.
- Run type checks, lint, tests, and production build.

## Definition of Done
- Verification suite passes.
- Static routes and generated assets are present in build output.
- New projects can be added with a repeatable documented workflow.
- Privacy/security limitations are explicit.

## Expected Artifacts
- README and contributor workflow.
- Validation/build scripts.
- Deployment configuration.
- Verification results.
